# Интернет-магазин «Web-Larёk»
«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и  отвечает за связь представления и данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

### Базовый код

#### Класс Component
Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент за отображение, которого он отвечает.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.  
`setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>`


#### Класс Api
Содержит в себе базовую логику отправки запросов.

Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
`baseUrl: string` - базовый адрес сервера  
`options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:  
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер  
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.  
`handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.

#### Класс EventEmitter
Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<string | RegExp, Set<Function>>)` -  хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
`on<T extends object>(event: EventName, callback: (data: T) => void): void` - подписка на событие, принимает название события и функцию обработчик.  
`emit<T extends object>(event: string, data?: T): void` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.  
`trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.

### Данные
Приложение разделено на два интерфейса: интерфейс товара и интерфейс покупателя.

#### Интерфейс товара (IProduct)
Товар, как главный элемент на сайте имеет описание, категорию и ценник. У некоторых товаров возможно отсутсвие цены. 

#### IProduct: 
`id: string` //уникальный индентификатор товара, тип данных - строка.\
`title: string` //название товара, тип данных - строка.\
`image: string` //ссылка (url) изображения товара, тип данных - строка.\
`category: string` // категория товаров, тип данных - строка.\
`price: number | null` //цена товара, может иметь значение null, тип данных - цифры или null.\
`description: string` //описание товара, тип данных - строка.\

#### Интерфейс покупателя (IBuyer)
В интерфейсе для покупателя присутствуют данные покупателя, необходимые для оформления заказа.

#### IBuyer
`payment: 'card' | 'cash' | ''` //выбор оплаты заказа, наличными или картой.\
`adress: string` //адрес доставки, тип данных - строка.\
`email: string` //электронная почта, тип данных - строка.\
`phone: string` //номер телефона, тип данных - строка.\

### Модель данных

#### Catalog (Каталог товаров)
Класс `Catalog` предназначен для управления каталогом товаров. Обеспечивает функционал:\
-Хранения списка товаров;\
-Поиска товара по идентификатору;\
-Добавления, обновления и удаления товаров;\
-Выбора текущего товара;\
-Получения информации о состоянии каталога (количество товаров, наличие конкретного товара);\
-Полной очистки каталога.\

Внутреннее состояние:\
Класс содержит два приватных поля:\
1. `products: IProduct[]` — массив всех товаров каталога. Инициализируется пустым массивом либо значениями из `initialProducts` при создании экземпляра.

2. `selectedProduct: IProduct | null` — выбранный товар (или `null`, если товар не выбран). Инициализируется как `null`.

Конструктор:\

`constructor(initialProducts: IProduct[] = [])`\
Создаёт экземпляр каталога с начальными данными или пустым массивом.

Параметры:\
`initialProducts: IProduct[]`(опционально) — массив товаров для начальной загрузки каталога. По умолчанию — пустой массив.

Методы:\
Геттеры и сеттеры\

```
get catalogData(): readonly IProduct[]
```

- Назначение: получить полный список товаров каталога.\
- Возвращает: неизменяемый массив `(readonly IProduct[])` товаров.\

```
set catalogData(products: IProduct[])
```

- Назначение: установить или обновить список товаров каталога.\
- Параметры: products: `IProduct[]` — массив товаров для установки.\
- Особенности: создаёт копию массива `([...products])`, проверяет, что `products` — массив. При ошибке выбрасывает исключение.\

```
get selectedProductData(): IProduct | null
```

- Назначение: получить выбранный товар.\
- Возвращает: объект `IProduct`, если товар выбран, или `null` в противном случае.\

```
set selectedProductData(product: IProduct | null)
```

Назначение: установить выбранный товар или сбросить выбор `(null)`.\
- Параметры: product: `IProduct` | `null` — товар для выбора или `null` для сброса.\
- Особенности: проверяет корректность товара через `isValidProduct`. При ошибке выбрасывает исключение.\

Публичные методы\

```
findProductById(productId: string): IProduct | null
```

- Назначение: найти товар по его уникальному идентификатору.\
- Параметры: `productId: string — ID` товара для поиска.\
- Возвращает: найденный объект `IProduct` или `null`, если товар не найден.\

```
addProduct(product: IProduct): void
```

- Назначение: добавить новый товар в каталог.\
- Параметры: `product: IProduct` — добавляемый товар.\
- Особенности: проверяет товар через `isValidProduct`. При ошибке выбрасывает исключение.\

```
removeProductById(productId: string): boolean
```

- Назначение: удалить товар по `ID`.\
- Параметры: `productId: string — ID` удаляемого товара.\
- Возвращает: `true`, если товар найден и удалён, иначе `false`.\
- Особенности: сбрасывает selectedProduct в `null`, если удалён выбранный товар.\
```
updateProduct(updatedProduct: IProduct): boolean
```
- Назначение: обновить данные существующего товара.\
- Параметры: `updatedProduct: IProduct` — обновлённые данные товара.\
- Возвращает: `true`, если товар найден и обновлён, иначе `false`.\
- Особенности: синхронизирует `selectedProduct`, если обновляется выбранный товар. \При ошибке выбрасывает исключение.

```
clearCatalog(): void
```

- Назначение: полностью очистить каталог и сбросить выбранный товар.\
- Действие: устанавливает products как пустой массив, `selectedProduct` как `null`.\

```
getProductCount(): number
```

- Назначение: получить количество товаров в каталоге.\
- Возвращает: число `(number)` — количество товаров.\

```
hasProduct(productId: string): boolean
```

- Назначение: проверить наличие товара в каталоге по ID.\
- Параметры: `productId: string` — ID товара для проверки.\
- Возвращает: `true`, если товар с указанным `id` существует, иначе `false`.\

Приватный метод\

```
private isValidProduct(product: IProduct): boolean
```

- Назначение: проверить корректность объекта товара.\
- Параметры: product: `IProduct` — проверяемый товар.\
- Возвращает: `true`, если:\
- товар существует `(!!product)`;\
`id` — строка `(typeof product.id === 'string')`;\
`id` не пустой `(product.id.trim().length > 0)`.\

Используется: внутри `addProduct`, `updateProduct` и `set` `selectedProductData`.\



#### Busket(Корзина)

Класс Basket предназначен для управления корзиной покупок. Обеспечивает функционал:\
- добавления товаров в корзину (с учётом уже имеющихся позиций);\
- обновления количества товаров;\
- удаления отдельных товаров или полной очистки корзины;\
- подсчёта общей стоимости и количества товаров;\
- проверки состояния корзины (пуста ли, есть ли конкретный товар);\
- получения информации о содержимом корзины.\


Внутреннее состояние\
Класс содержит два поля:\
1. `items: BasketItem[]` — приватное поле, массив элементов корзины. Каждый элемент имеет тип `BasketItem` (товар + количество). Инициализируется пустым массивом.\
2. `maxQuantityPerItem: number` — приватное неизменяемое поле, задающее максимальное допустимое количество единиц одного товара в корзине. По умолчанию — 99.\

Конструктор\
Конструктор по умолчанию `(constructor())` не требует параметров и инициализирует корзину с пустым массивом товаров и стандартным лимитом в 99 единиц на товар.

Методы\
Геттеры\

```
get products(): readonly BasketItem[]
```

- Назначение: получить копию массива элементов корзины.\
- Возвращает: неизменяемый массив `(readonly BasketItem[])` элементов корзины.Предотвращает прямое изменение внутреннего состояния.\

```
get totalQuantity(): number
```

- Назначение: подсчитать общее количество товаров в корзине (с учётом количества каждого товара).\
- Возвращает: `число (number)` — сумма количеств всех товаров.\

```
get totalPrice(): number
```

- Назначение: подсчитать общую стоимость корзины (цена × количество для каждого товара). Если цена не указана, считается как 0.\
- Возвращает: число `(number)` — итоговая стоимость.\

```
get isEmpty(): boolean
```

- Назначение: проверить, пуста ли корзина.\
- Возвращает: `true`, если корзина пуста `(items.length === 0)`, иначе `false`.\

```
get itemCount(): number
```

-Назначение: получить количество уникальных товаров (позиций) в корзине.\
-Возвращает: число `(number)` — количество элементов в массиве `items`.\

Публичные методы\

```
addItemInBasket(product: IProduct, quantity: number = 1): void
```

- Назначение: добавить товар в корзину. Если товар уже есть, увеличивает его количество.\
- Параметры:\
`product: IProduct` — добавляемый товар.\
`quantity: number` (опционально) — количество товара. По умолчанию — 1.\
- Особенности:\
проверяет, что `quantity ≥ 1`;\
проверяет, что итоговое количество не превышает `maxQuantityPerItem`;\
при ошибке выбрасывает исключение.\

```
updateQuantity(productId: string, quantity: number): void
```

-Назначение: обновить количество товара в корзине.
-Параметры:
`productId: string — ID` товара для обновления.
`quantity: number` — новое количество товара.
-Особенности:
проверяет, что `quantity` в диапазоне `[1, maxQuantityPerItem]`;
выбрасывает ошибку, если товар не найден или количество некорректно.

```
removeItem(productId: string): void
```

- Назначение: удалить товар из корзины по `ID`.\
- Параметры: `productId: string — ID` удаляемого товара.\
- Действие: фильтрует массив `items`, исключая товар с указанным `ID`.\

```
clear(): void

```

- Назначение: полностью очистить корзину.\
- Действие: устанавливает `items` как пустой массив.\

```
hasProduct(productId: string): boolean
```

Назначение: проверить наличие товара в корзине по `ID`.\
Параметры: `productId: string — ID` товара для проверки.\
Возвращает: `true`, если товар с указанным `id` есть в корзине, иначе `false`.\

```
findItem(productId: string): BasketItem | undefined
```

- Назначение: найти элемент корзины по `ID` товара.\
- Параметры: `productId: string — ID` товара для поиска.\
- Возвращает: объект `BasketItem`, если найден, или `undefined`, если не найден.\



#### Buyer (форма покупателя)
Класс `Buyer` предназначен для управления данными покупателя и их валидации. Обеспечивает функционал:\
- хранения данных покупателя;\
- получения и обновления данных (частичного);\
- очистки данных покупателя;\
- валидации данных с формированием подробных сообщений об ошибках;\
- проверки корректности заполнения всех данных.\

Внутреннее состояние\
Класс содержит одно приватное поле:\
- `buyer: IBuyer` — объект с данными покупателя. Инициализируется копией `emptyBuyerData` (пустые данные, соответствующие интерфейсу `IBuyer`).

Конструктор\

`constructor()` Создаёт экземпляр класса с пустыми данными покупателя по умолчанию.

Действие: инициализирует `this.buyer` через `{ ...emptyBuyerData }`, гарантируя соответствие типу `IBuyer`.\

Методы\
Геттеры и сеттеры\

```
get buyerData(): IBuyer
```

- Назначение: получить текущие данные покупателя.\
- Возвращает: объект типа `IBuyer` с текущими данными покупателя.\

```
set buyerData(buyer: Partial<IBuyer>)
```

- Назначение: обновить данные покупателя (частично).\
- Параметры: `buyer: Partial<IBuyer>` — объект с полями для обновления (не обязательно все поля `IBuyer`).\
- Особенности:\
использует слияние объектов `({ ...this.buyer, ...buyer })` для частичного обновления;
сохраняет значения существующих полей, если они не переданы в `buyer`.

Публичные методы\

```
clearBuyerData(): void
```

- Назначение: сбросить данные покупателя к начальному (пустому) состоянию.\
- Действие: устанавливает `this.buyer` как копию `emptyBuyerData`.\

```
validateForm(): IErrorsBuyer
```

- Назначение: проверить корректность заполнения данных покупателя и сформировать объект с ошибками.\
- Возвращает: объект типа `IErrorsBuyer`, где:\
ключи — имена полей `(address, phone, email, payment)`;\
значения — текстовые сообщения об ошибках (если поле некорректно), либо поле отсутствует (если ошибок нет).\

- Логика валидации:
`Адрес`: не может быть пустым `(!this.buyer.address.trim() → ошибка)`.
`Телефон`: не может быть пустым; должен соответствовать регулярному выражению `/^[\d\s\-\+\(\)]{10,}$/` (минимум 10 цифр, пробелы, дефисы, скобки, плюсы).
`Email`: не может быть пустым; должен соответствовать базовому шаблону email `/^[^\s@]+@[^\s@]+\.[^\s@]+$/`.
`Способ оплаты`: не может быть `null` `(this.buyer.payment === null → ошибка)`.

```
isValid(): boolean
```

- Назначение: быстро проверить, корректны ли все данные покупателя (нет ошибок валидации).\
- Возвращает: true, если ошибок нет `(Object.keys(this.validateForm()).length === 0)`, иначе `false`.\
- Особенности: вызывает `validateForm()` и проверяет длину полученного объекта ошибок.\

### CatalogService (Слой коммуникации)
Класс `CatalogService` — сервисный класс для взаимодействия с `API` в контексте каталога товаров и обработки заказов. Обеспечивает функционал:\
- загрузки данных каталога товаров;\
- отправки заказов на сервер;\
- предварительной валидации данных заказа перед отправкой.\

Внутреннее состояние\
Класс содержит два поля:\
- `api: IApi` — приватное поле, экземпляр `API‑клиента`, используемый для выполнения `HTTP‑запросов`. Передаётся через конструктор.\
- `timeoutMs: number` — приватное неизменяемое поле, задающее таймаут запросов в миллисекундах. По умолчанию — 10 000 мс (10 секунд).\

Конструктор\

`constructor(api: IApi)` Создаёт экземпляр сервиса с указанным `API‑клиентом`.

Параметры:\
`api: IApi` — реализация интерфейса `IApi` для выполнения `HTTP‑запросов` (`get`, `post` и т. д.).

Методы\
Публичные методы\

```
async getCatalogProducts(): Promise<ICatalogFromApi>
```

- Назначение: загрузить данные каталога товаров с сервера.\
- Возвращает: Promise<ICatalogFromApi> — промис с данными каталога в формате ICatalogFromApi.\
- Логика работы:\
Выполняет GET‑запрос к эндпоинту `/product через this.api.get`.
При успехе возвращает ответ сервера.
При ошибке:
логирует ошибку в консоль `(console.error)`;
выбрасывает обобщённое сообщение о проблеме с подключением.
Ошибки:
`«Не удалось загрузить каталог товаров. Проверьте подключение к интернету.»`

```
async postOrder(value: IOrder): Promise<IOrderResult>
```

- Назначение: отправить заказ на сервер.\
- Параметры: `value: IOrder` — объект заказа для отправки.\
- Возвращает: `Promise<IOrderResult>` — промис с результатом оформления заказа.\
- Логика работы:\
Предварительно валидирует данные заказа через `validateOrderData`.\
Выполняет `POST‑запрос` к эндпоинту `/order` с телом `value`.\
При успехе возвращает результат сервера.\
При ошибке:\ 
логирует ошибку;\ 
для статуса 400 возвращает специфичное сообщение;\
в остальных случаях — общее сообщение об ошибке.\

- Ошибки:\
`«Некорректные данные заказа. Проверьте заполнение всех полей.»` (для статуса 400);\
`«Произошла ошибка при оформлении заказа.»` (для других ошибок).\
