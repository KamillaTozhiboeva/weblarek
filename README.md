# Интернет-магазин «Web-Larёk»

«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и отвечает за связь представления и данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

### Базовый код

#### Класс Component

Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент за отображение, которого он отвечает.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.  
`setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>`

#### Класс Api

Содержит в себе базовую логику отправки запросов.

Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
`baseUrl: string` - базовый адрес сервера  
`options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:  
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер  
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.  
`handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.

#### Класс EventEmitter

Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<string | RegExp, Set<Function>>)` - хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
`on<T extends object>(event: EventName, callback: (data: T) => void): void` - подписка на событие, принимает название события и функцию обработчик.  
`emit<T extends object>(event: string, data?: T): void` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.  
`trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.

### Данные

Приложение разделено на два интерфейса: интерфейс товара и интерфейс покупателя.

#### Интерфейс товара (IProduct)

Товар, как главный элемент на сайте имеет описание, категорию и ценник. У некоторых товаров возможно отсутсвие цены.

#### IProduct:

`id: string` уникальный индентификатор товара, тип данных - строка.  
`title: string` название товара, тип данных - строка.  
`image: string` ссылка (url) изображения товара, тип данных - строка.  
`category: string` категория товаров, тип данных - строка.  
`price: number | null` цена товара, может иметь значение null, тип данных - цифры или null.  
`description: string` описание товара, тип данных - строка.  

#### Интерфейс покупателя (IBuyer)

В интерфейсе для покупателя присутствуют данные покупателя, необходимые для оформления заказа.

#### IBuyer

`payment: 'card' | 'cash' | ''` выбор оплаты заказа, наличными или картой.  
`adress: string` адрес доставки, тип данных - строка.  
`email: string` электронная почта, тип данных - строка.  
`phone: string` номер телефона, тип данных - строка.  

### Модель данных

Эти классы отвечают за хранение состояния приложения и бизнес-логику (валидация, расчет стоимости). Они полностью изолированы от DOM-элементов и API.

#### Catalog (Каталог товаров)

Предназначен для управления каталогом товаров.

Поля:
`products: IProduct[]` — массив товаров.  
`selectedProduct: IProduct | null` — товар, выбранный для предпросмотра.  

Геттеры/Сеттеры:

`catalogData` — установка и получение списка товаров.  
`electedProductData` — выбор товара для просмотра или сброс выбора.  

Методы:
`findProductById(id: string): IProduct | null` — поиск товара по уникальному идентификатору.  
`clearCatalog(): void` — очистка данных.  

#### Busket(Корзина)

Отвечает за хранение и управление списком товаров, которые пользователь выбрал для покупки.  

Поля:
`_items: IProduct[]` — массив объектов товаров, добавленных в корзину.  

Геттеры:
`items: readonly IProduct[]` — возвращает неизменяемый список товаров в корзине.  
`totalPrice: number` — вычисляет суммарную стоимость всех товаров в корзине.  
`itemCount: number` — возвращает общее количество товаров в корзине.  

Методы:
`addItem(product: IProduct): void` — добавляет товар в корзину.  
`removeItem(productId: string): void` — удаляет товар из корзины по его уникальному идентификатору.  
`clear(): void` — полностью очищает корзину.  
`hasProduct(productId: string): boolean` — проверяет, находится ли товар с данным ID в корзине.  

#### Buyer (форма покупателя)

Отвечает за хранение персональных данных клиента и проверку корректности заполнения форм заказа.  

Поля:
`_buyer: IBuyer`— объект с данными клиента.  

Геттеры/Сеттеры:
`buyerData` — позволяет получить текущие данные или обновить их через частичное слияние.  

Методы:
`validateForm(): IErrorsBuyer` — проверяет все поля на заполнение. Если поле пустое, формирует текст ошибки. Используется для определения готовности заказа к отправке.  

`clearBuyerData(): void` — сброс всех данных покупателя.  

### CatalogService (Слой коммуникации)

Обеспечивает взаимодействие приложения с сервером.  

Конструктор:  
`constructor(api: IApi)` — принимает экземпляр класса для работы с сетевыми запросами.  

Методы:  
`getCatalogProducts(): Promise<ICatalogFromApi>` — получает полный список товаров от сервера.  
`postOrder(order: IOrder): Promise<IOrderResult>` — отправляет данные заказа на сервер и возвращает результат покупки.  

### Типы данных

Все ключевые типы и интерфейсы описаны в файле src/types/index.ts. Основные из них:
`IProduct` — структура товара.  
`IBuyer` — структура данных покупателя.  
`IErrorsBuyer` — динамический тип ошибок, построенный на основе IBuyer с использованием Partial<Record<keyof IBuyer, string>>.  
`IOrder` — объект заказа, объединяющий корзину и данные клиента.  


### Компоненты отображения (View)

Все компоненты отображения наследуются от базового класса `Component<T>` и принимают в конструктор целевой DOM-элемент.

#### Класс `Page`
Управляет основными глобальными элементами интерфейса на главной странице (каталог, счетчик корзины, блокировка прокрутки).
**Конструктор**:
`constructor(container: HTMLElement, events: IEvents)` — принимает контейнер страницы и инициализирует обработчики событий.
**Поля**:
`_counter: HTMLElement` — элемент счетчика товаров в иконке корзины.
`_catalog: HTMLElement` — контейнер для отрисовки карточек товаров.
`_wrapper: HTMLElement` — главный элемент-обертка страницы (используется для блокировки скролла).
**Методы (сеттеры)**:
`set counter(value: number): void` — обновляет текстовое значение счетчика корзины.
`set catalog(cards: HTMLElement[]): void` — заменяет содержимое контейнера каталога новыми элементами карточек.
`set locked(value: boolean): void` — добавляет/удаляет класс блокировки страницы при открытии модальных окон.

#### Класс `Card`
Базовый компонент для отображения карточки товара. Имеет несколько подклассов под конкретные контексты (каталог, превью, корзина).
**Конструктор**:
`constructor(blockName: string, container: HTMLElement, actions?: ICardActions)` — принимает имя блока для поиска элементов, контейнер карточки и объект с колбэками (например, `onClick`).
**Поля**:
`_title: HTMLElement` — элемент заголовка товара.
`_price: HTMLElement` — элемент отображения стоимости.
`_category?: HTMLElement` — элемент категории (опционально).
`_image?: HTMLImageElement` — элемент изображения товара (опционально).
`_button?: HTMLButtonElement` — кнопка действия (опционально).
**Методы (сеттеры/геттеры)**:
`set id(value: string): void` / `get id(): string` — управляет идентификатором товара в data-атрибутах.
`set title(value: string): void` / `get title(): string` — управляет заголовком.
`set image(value: string): void` — устанавливает полный путь к картинке с использованием `CDN_URL`.
`set category(value: string): void` — устанавливает текст категории и навешивает соответствующий CSS-класс.
`set price(value: number | null): void` — выводит цену (или текст "Бесценно" с деактивацией кнопки, если цена `null`).

#### Класс `Modal`
Реализует управление модальными окнами (открытие, закрытие, подстановка контента).
**Конструктор**:
`constructor(container: HTMLElement, events: IEvents)` — принимает контейнер модального окна, находит кнопку закрытия и вешает слушатели на клик по оверлею и кнопке «X».
**Поля**:
`_closeButton: HTMLButtonElement` — кнопка закрытия модального окна.
`_content: HTMLElement` — контейнер для динамического контента внутри окна.
**Методы**:
`set content(value: HTMLElement): void` — заменяет текущее содержимое окна новым элементом.
`open(): void` — открывает модальное окно и генерирует событие `modal:open`.
`close(): void` — закрывает окно, очищает контент и генерирует событие `modal:close`.
`render(data: IModalData): HTMLElement` — заполняет контент и открывает окно.

#### Класс `BasketView`
Отображает список элементов в корзине, сквозной порядковый номер и итоговую сумму.
**Конструктор**:
`constructor(container: HTMLElement, events: IEvents)` — принимает шаблон корзины, вешает слушатель на кнопку оформления.
**Поля**:
`_list: HTMLElement` — контейнер (список `<ul>` или `<div>`) для карточек товаров.
`_total: HTMLElement` — элемент для вывода итоговой стоимости.
`_button: HTMLButtonElement` — кнопка перехода к оформлению заказа.
**Методы**:
`set items(items: HTMLElement[]): void` — рендерит переданный массив элементов товаров в корзине или выводит заглушку, если корзина пуста.
`set totalPrice(total: number): void` — обновляет значение итоговой стоимости и управляет доступностью кнопки (блокирует, если `total === 0`).

#### Класс `Form<T>`
Универсальный компонент для управления формами ввода и вывода ошибок.
**Конструктор**:
`constructor(container: HTMLFormElement, events: IEvents)` — инициализирует форму, вешает слушатели на ввод (`input`) и отправку (`submit`).
**Поля**:
`_submit: HTMLButtonElement` — кнопка отправки формы.
`_errors: HTMLElement` — элемент для вывода сообщений об ошибках валидации.
**Методы**:
`protected onInputChange(field: keyof T, value: string): void` — инициирует событие изменения конкретного поля в шине данных.
`set valid(value: boolean): void` — управляет активностью кнопки отправки (`disabled`).
`set errors(value: string): void` — выводит текст ошибок на экран.
`render(state: Partial<T> & IFormState): HTMLElement` — обновляет состояние полей, валидности и ошибок формы.

#### Класс `Success`
Компонент финального окна успешного завершения покупки.
**Конструктор**:
`constructor(container: HTMLElement, actions?: ISuccessActions)` — принимает контейнер и объект с действием на закрытие/клик по кнопке.
**Поля**:
`_close: HTMLButtonElement` — кнопка «За покупками!».
`_total: HTMLElement` — элемент текста для вывода информации о списанных синапсах.
**Методы**:
`set totalPrice(value: number): void` — выводит фразу о том, сколько синапсов было успешно оплачено.

### Основные события в приложении (EventEmitter)

Вместо прямых вызовов между моделями и отображением используется событийно-ориентированный подход. По требованию архитектуры, все формы и модели генерируют **события изменения данных**, а не напрямую события валидации.

| Название события | Описание |
| :--- | :--- |
| `items:changed` | Инициируется моделью `Catalog` при изменении или загрузке списка товаров. На него реагирует `Page` для перерисовки витрины. |
| `card:select` | Возникает при клике на карточку товара. Передает выбранный объект товара для открытия превью. |
| `preview:changed` | Возникает в модели данных при изменении активного товара для превью. Приводит к рендерингу карточки внутри `Modal`. |
| `product:toBasket` | Срабатывает при клике на кнопку добавления/удаления товара в окне превью. |
| `basket:changed` | **Событие изменения данных корзины**. Генерируется моделью `Basket` при любом добавлении или удалении товара. Компонент `BasketView` пересчитывает общую сумму, обновляет список и счетчик на главной странице. |
| `basket:open` | Клик по иконке корзины на главной странице. Вызывает открытие модального окна с корзиной. |
| `order:open` | Клик по кнопке «Оформить» в корзине. Открывает первый шаг формы заказа (выбор оплаты и адреса). |
| `buyer:changed` | **Событие изменения данных покупателя**. Генерируется бизнес-моделью `Buyer` при вводе данных в любое поле ввода (`order` или `contacts`). Модель принимает новые данные, обновляет своё состояние, проводит внутреннюю валидацию и следом уведомляет систему через `formErrors:change`. |
| `formErrors:change` | Возникает в результате внутренней проверки бизнес-модели. Передает объект с ошибками, на основе которого формы `Order` или `Contacts` блокируют кнопки сабмита. |
| `order:submit` | Сабмит первой формы. Переводит пользователя на второй шаг (ввод контактов). |
| `contacts:submit` | Финальный сабмит формы контактов. Запускает процесс отправки данных на сервер через `Api`. |
| `modal:open` / `modal:close` | Технические события модального окна, по которым `Page` включает или отключает блокировку скролла страницы. |
